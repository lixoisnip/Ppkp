# A03/A04 top packet-function static trace

## Почему выбраны `0x889F` (A04) и `0x8904` (A03)

Эти точки уже выделены в предыдущем шаге как top packet-builder candidates (парные функции в окне A03/A04 с близкой структурой control-flow и XDATA-обращениями к служебным буферам/селектору). В этом шаге выполнен только детальный **статический** trace по инструкциям и блокам, без заявлений о полном восстановлении wire-format.

## Состав blocks внутри функций

- `A04_28.PZU:0x889F` blocks:
  - `0x889F, 0x88A5, 0x88AE, 0x88B4, 0x88B7, 0x88B8, 0x88CA, 0x88D1, 0x88DB, 0x88F4, 0x88F7, 0x88F9, 0x8900`
- `A03_26.PZU:0x8904` blocks:
  - `0x8904, 0x890A, 0x8913, 0x8919, 0x891C, 0x891D, 0x892F, 0x8936, 0x8940, 0x8959, 0x895C, 0x895E, 0x8965`

Для обеих функций в trace попало по 44 инструкции (всего 88 строк данных).

## Какие XDATA-адреса встречаются

- `A04_28.PZU:0x889F`:
  - `0x329C, 0x329D, 0x32A0, 0x32A1, 0x4DAC, 0x500C, 0x500D`
- `A03_26.PZU:0x8904`:
  - `0x3298, 0x3299, 0x329C, 0x329D, 0x4DB6, 0x4FD7, 0x4FD8`

### Отдельно по ключевым адресам

- `0x329C`: есть в обеих функциях.
- `0x329D`: есть в обеих функциях.
- `0x5003..0x5010`:
  - в `A04:0x889F` есть попадания (`0x500C`, `0x500D`);
  - в `A03:0x8904` в этом trace попаданий в диапазон нет.
- `0x3298`:
  - есть в `A03:0x8904`;
  - в `A04:0x889F` в этом trace не зафиксирован.
- `0x32A2`:
  - в обеих функциях в текущем trace не зафиксирован.

## MOVC/code-table и переходы

- MOVC/code-table hits внутри выбранных blocks: в текущем trace для обеих функций **не найдено**.
- По `call_xref.csv` внутри blocks зафиксированы `SJMP`:
  - `A04:0x889F`: `0x88E1 -> 0x8906`, `0x88FE -> 0x8906`;
  - `A03:0x8904`: `0x8946 -> 0x896B`, `0x8963 -> 0x896B`.

## Сравнение A04 `0x889F` и A03 `0x8904`

- Структура сопоставима: одинаковый объём trace (44 инструкции) и близкий паттерн внутренних short-jump.
- Обе функции трогают пару `0x329C/0x329D`, что поддерживает гипотезу про общий pipeline-узел.
- Различия по XDATA-окнам:
  - A04-ветка в этом фрагменте уже пишет/читает внутри `0x5003..0x5010`;
  - A03-ветка в аналогичном участке показывает обращения к `0x3298/0x3299` и другим локальным адресам (`0x4FD7/0x4FD8`).

## Ограничение текущего шага

Это **trace статического анализа** по уже построенным индексам (`disassembly_index`, `basic_block_map`, `xdata_confirmed_access`, `code_table_candidates`, `call_xref`). Он не является runtime-доказательством фактического wire-format поведения на стенде.
